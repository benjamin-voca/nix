## Architecture Notes

### Secret Resolution Flow
```
Machine registry (machines/default.nix)
  → consumer.nix reads machine.secrets.files
  → typed-secrets.nix validates requiredSecrets exist in layered files
  → Generates sops.secrets attrset per-host
  → Injected as extraModule into mkClusterHost
```

### Layered Lookup Order (later wins)
```
secrets/shared.yaml → secrets/roles/<role>.yaml → secrets/hosts/<host>.yaml
```

### What's NOT in requiredSecrets (but in SOPS file)
These secrets exist in `secrets/roles/backbone.yaml` but are NOT in the backbone role's `requiredSecrets`. They won't be auto-mounted:
- `cloudflare-www-edit-workers-token`
- `grafana-admin-password`
- `openclaw-beni-discord-id`

- `orkestr-resend-api-key`
- `quadpacienti-drilon-resend-key`

Some of these are checked by `k8s-secrets-inject.nix` with `if [ -f ]` guards — those code paths will simply skip. Add them to `requiredSecrets` when needed.

### Key Design Decision: cloudflared-credentials path
The original manual config used `path = "/run/secrets/cloudflared-credentials.json"`. The typed-secrets library generates `/run/secrets/cloudflared-credentials` (no extension). The cloudflared preStart script was updated to match. This is a **breaking change** — the secret file path changes on the deployed host.
